# Contributing

Этот репозиторий создан в учебных целях, критика или предложения к оформлению приветствуются.

## Процесс работы

1. Создайте форк репозитория.
2. Создайте отдельную ветку для изменений.
3. Внесите необходимые изменения.
4. Убедитесь, что изменения не содержат чувствительных данных.
5. Создайте Pull Request с описанием выполненной работы.

## Именование веток

Рекомендуется использовать следующие шаблоны:

```text
feature/add-readme-section
feature/update-examples
fix/markdown-typo
fix/gitignore-rules
docs/update-lab
```

## Требования к изменениям

- Не публиковать приватные SSH-ключи.
- Не публиковать токены, пароли и API-ключи.
- Не публиковать корпоративные данные.
- Сохранять читаемую структуру Markdown-документов.
- Использовать понятные сообщения коммитов.

## Рекомендации по коммитам

Примеры хороших сообщений:

```text
Add README project description
Update SECURITY policy
Fix markdown formatting
Add gitconfig example
```

Также можно использовать коды задач, например если это Jira: APPSUP-1234

## Code Review

Перед отправкой Pull Request рекомендуется самостоятельно проверить:

- корректность Markdown-разметки;
- отсутствие секретов и конфиденциальных данных;
- актуальность документации;
- работоспособность примеров команд.

## Вопросы и предложения

Если вы нашли ошибку или хотите предложить улучшение, создайте Issue или Pull Request с подробным описанием изменений.
