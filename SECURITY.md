# Security Policy

## Reporting a Vulnerability

Если вы обнаружили потенциальную уязвимость, пожалуйста, не публикуйте её сразу в открытом Issue.

Для сообщения об уязвимости используйте следующий контакт:

```text
your@email.com
```

В сообщении желательно указать:

- описание проблемы;
- шаги для воспроизведения;
- предполагаемое влияние;
- возможные рекомендации по исправлению.

## Scope

В рамках данного репозитория считаются чувствительными следующие данные:

- приватные SSH-ключи;
- API-токены;
- пароли;
- секреты CI/CD;
- корпоративные адреса и внутренние домены.

Такие данные не должны попадать в репозиторий ни в коде, ни в документации, ни на скриншотах.

## Security Best Practices

При работе с Git рекомендуется:

- использовать SSH вместо передачи паролей;
- регулярно обновлять ключи доступа;
- не хранить секреты в репозитории;
- использовать `.gitignore` для исключения локальных конфигураций;
- проверять изменения перед каждым коммитом командой:

```bash
git diff --staged
```
